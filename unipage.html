Given the assignment number one of Gianluca Levi for the course "Programming for Computational Chemistry" (see here) here I write my short, funky report.

TASK 1 - Extract the number of atoms from an .xyz file without opening it!

What do we know?

  • First line contains the number of atoms
  • I have a dog and two cats
  • One of these is irrelevant for the task
Then we propose three approaches:
  1. The Peeker
  2. The verbose
  3. The "I have no idea how this command works but it's fine"

1. The Peeker

First we use $head filename.xyz to print in the shell the first 10 lines, then we look at the number of atoms and write it on a file calling $grep -w "[number of atoms]" filename.xyz>ass1.txt, where -w serves the purpose of selecting only the lines where the number appears as an entire string and not a substring, while ass1.txt is our target file. Simple but there's little to no automation.

2. The verbose

Here we avoid peeking at the number and propose a more general approach, supposing that we know the line we are looking for is the nth one. We just call $head -[line number] filename.xyz|tail -1>ass1.txt so to get the first [line number] lines of the file and from those we take the last line. We use the pipe operator | to send the output of head as the input of tail. Cool but I imagine that it can become unefficient with big files.

3. The "I have no idea how this command works but it's fine"

Just call $sed -n '[line number]p' filename.xyz>ass1.txt, where -n suppresses the output of the whole content, p prints the [line number] lines.

TASK 2 - Extract the coordinates of every nitrogen atom from molecule1.xyz!

Easier than the former task, just use $grep "N" molecule1.xyz>>ass1.txt so to append the new strings to the file where we already wrote the answer to the previous task.

TASK 3 - Describe the differences between molecule1.xyz and molecule2.xyz using vimdiff

Here we just use vimdiff and list the differences highlighted.

To condense all the results in a neat interactive way, I wrote a simple bash executable that you can find in my directory that does on the fly the first two tasks. I would insert down here a button or an interactive window to execute it but I don't have the necessary skill yet.
